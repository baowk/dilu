- **`cmd/`** - Command line tool entry (gen, start, version, etc.)

- **`internal/`** - Core business logic layer (replaces old modules directory)
  - `bootstrap/` - Application initialization
  - `tools/` - Code generator core implementation
    - `apis/` - API interface layer
    - `service/` - Business logic layer
    - `router/` - Router configuration layer
    - `repository/` - Data access layer
      - `model/` - Model layer
      - `query/` - Query layer
  - `sys/` - System management module (auto-generated + manual extension)
    - `repository/` - Data access layer (Model, Query, generated by gen)
      - `model/` - Model layer (*.gen.go)
      - `query/` - Query layer (*.gen.go)
    - `apis/` - API interface layer (manually created)
    - `service/` - Business logic layer (manually created, core workspace)
    - `router/` - Router configuration layer (manually created)
- **`resources/`** - Configuration files (different environments)
- **`scripts/`** - Scripts (database initialization, migration, etc.)
- **`temp/`** - Temporary files (logs, SQLite database, runtime files)

### ⚠️ Important Principles

When developing with this framework, you must follow these core principles:

1. **Generated Code Must Not Be Modified**: Repository/Model/Query layer code generated by gen command is strictly prohibited from manual modification
2. **Business Logic Concentrated in Service Layer**: All business logic must be implemented in the Service layer
3. **Clear Responsibility Boundaries**: Repository layer handles data persistence, Service layer handles business logic, Api layer encapsulates HTTP interfaces
4. **Safe Iteration Process**: Modify database → Regenerate code → Adapt in Service layer

Detailed specifications and practices will be explained in subsequent sections.

```go
    user, err := query.SysUser.Where(query.SysUser.ID.Eq(userID)).First()
    if err != nil {
        return err
    }
    
    // Business logic: Calculate bonus based on current level
    bonusRate := getBonusRateByLevel(user.Level)
    actualPoints := int32(float64(points) * bonusRate)
    
    // Update points
    _, err = query.SysUser.Where(query.SysUser.ID.Eq(userID)).Update(
        query.SysUser.Points.Value(user.Points + actualPoints),
    )
    return err
}

// DeductPoints deduct user points
func (s *UserPointsService) DeductPoints(userID int32, points int32, reason string) error {
    if points <= 0 {
        return fmt.Errorf("points must be greater than 0")
    }
    
    user, err := query.SysUser.Where(query.SysUser.ID.Eq(userID)).First()
    if err != nil {
        return err
    }
    
    if user.Points < points {
        return fmt.Errorf("insufficient points")
    }
    
    _, err = query.SysUser.Where(query.SysUser.ID.Eq(userID)).Update(
        query.SysUser.Points.Value(user.Points - points),
    )
    return err
}

// LevelUp user level up
func (s *UserPointsService) LevelUp(userID int32) error {
    user, err := query.SysUser.Where(query.SysUser.ID.Eq(userID)).First()
    if err != nil {
        return err
    }
    
    // Business logic: Check if level up condition is met
    requiredPoints := getRequiredPointsForLevel(user.Level)
    if user.Points < requiredPoints {
        return fmt.Errorf("insufficient points, need %d points to reach level %d", requiredPoints, user.Level+1)
    }
    
    // Level up
    _, err = query.SysUser.Where(query.SysUser.ID.Eq(userID)).Update(
        query.SysUser.Level.Value(user.Level + 1),
    )
    return err
}

func getBonusRateByLevel(level int32) float64 {
    // Level bonus logic
    switch level {
    case 1:
        return 1.0
    case 2:
        return 1.1
    case 3:
        return 1.2
    default:
        return 1.5
    }
}

func getRequiredPointsForLevel(level int32) int32 {
    // Points required for level up logic
    return level * 1000
}
```

**Router Layer Example** (`sys_user.go`):
```go
package router

import (
    "github.com/gin-gonic/gin"
    jwt "github.com/appleboy/gin-jwt/v2"
    "dilu/internal/sys/apis"
    "dilu/common/middleware"
)

type SysUserRouter struct{}

func (r *SysUserRouter) InitSysUserRouter(group *gin.RouterGroup, authMiddleware *jwt.GinJWTMiddleware) gin.IRoutes {
    // User management route group
    userRouter := group.Group("/user")
    userRouter.Use(middleware.Logger()) // Logger middleware
    
    // Public routes (no authentication required)
    publicRouter := userRouter.Group("")
    {
        publicRouter.POST("/register", apis.SerSysUserApi.Register)      // User registration
        publicRouter.POST("/login", authMiddleware.LoginHandler)         // User login
    }
    
    // Protected routes (authentication required)
    protectedRouter := userRouter.Group("")
    protectedRouter.Use(authMiddleware.MiddlewareFunc())
    {
        protectedRouter.GET("", apis.SerSysUserApi.GetUserList)          // User list
        protectedRouter.GET("/:id", apis.SerSysUserApi.GetUserByID)      // User details
        protectedRouter.PUT("", apis.SerSysUserApi.UpdateUser)           // Update user
        protectedRouter.DELETE("/:id", apis.SerSysUserApi.DeleteUser)    // Delete user
    }
    
    return userRouter
}
```

## ❓ FAQ

### Q1: What if AI-generated code doesn't follow project standards?

**A:** 
1. Clearly state project standards and conventions in prompts
2. Provide example code as reference
3. Use AI's code review feature to check generated code
4. Build project code templates for AI reference

### Q2: How to make AI understand business logic?

**A:**
1. Provide detailed business context
2. Use domain-specific terminology
3. Provide business flow diagrams and data structures
4. Guide AI step-by-step through complex logic

### Q3: What to do if AI-generated code has bugs?

**A:**
1. Use AI debugging features to locate issues
2. Provide detailed error messages and stack traces
3. Let AI analyze possible causes
4. Combine with manual review to ensure code quality

### Q4: How to balance AI-generated and handwritten code?

**A:**
- **Suitable for AI generation**:
  - Repetitive CRUD code
  - Standardized API interfaces
  - Basic unit tests
  - Documentation comments
  
- **Suitable for handwritten code**:
  - Core business logic
  - Complex algorithm implementations
  - Performance-critical code
  - Security-related code

### Q5: Can AI replace human developers?

**A:** 
AI is a powerful tool but cannot completely replace humans:
- ✅ AI excels at: Rapid generation, pattern matching, repetitive tasks
- ❌ AI limitations: Business understanding, architecture design, creative thinking
- 💡 Best practice: Human-AI collaboration, AI for efficiency, human for quality control
